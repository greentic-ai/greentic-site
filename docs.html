Quickstart

Install the CLI, login, and create your first worker.

curl -fsSL https://get.greentic.ai/install.sh | sh
greentic login
greentic worker new my-first-worker
curl -H "authorization: Bearer $TOKEN" https://api.greentic.ai/health
import {{ Greentic }} from "@greentic/sdk";
const g = new Greentic(process.env.KEY);
use greentic::Client;

Concepts

  • Workers — WASM units that execute tasks.
  • Flows — Orchestrations using MCP.
  • Skills/Tools — Capabilities exposed by plugins.
  • Channels — In/Out surfaces (email, Teams, webhooks).
  • Secrets — Encrypted credentials with least privilege.

Deploy guides

GitHub Pages

  1. Push this repo to main
  2. Settings → Pages → Source: Deploy from a branch, Branch: main, / (root)
  3. Optional: add CNAME greentic.ai

Cloudflare Pages

Point to this repo; no build step needed.
